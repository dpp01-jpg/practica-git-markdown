# ¿Qué es una Pull Request?

Una Pull Request (PR) es una solicitud para fusionar cambios de una rama a otra en un repositorio Git.

## ¿Para qué sirve?

- Revisar el código antes de mezclarlo con main.
- Comentar cambios.
- Controlar mejor el historial.
- Trabajar en equipo sin sobrescribir el trabajo de otros.

## Ventajas
- Mejora la calidad del código.
- Facilita la colaboración.
- Permite detectar errores antes de fusionar.

## Desventajas
- Puede ser lenta si requiere mucha revisión.
- Genera conflictos si muchas personas cambian los mismos archivos.
